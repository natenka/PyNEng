### Аутентификация на GitHub

Для того, чтобы начать работать с GitHub, надо на нём [зарегистрироваться](https://github.com/join). Для безопасной работы, лучше использовать аутентификацию по ключам SSH.

> Подобная [инструкция](https://help.github.com/articles/connecting-to-github-with-ssh/) на GitHub

Генерация нового SSH-ключа (используйте e-mail, который привязан к GitHub):

```shellsession
$ ssh-keygen -t rsa -b 4096 -C "example@example.com"
```

На всех вопросах достаточно нажать Enter (более безопасно использовать ключ с passphrase, но удобнее без него (если просто нажать Enter при запросе) тогда при использовании репозитория не будет постоянно спрашиваться пароль).

Запуск SSH-агента:

```shellsession
$ eval "$(ssh-agent -s)"
```

Добавить ключ в SSH-агент:

```shellsession
$ ssh-add ~/.ssh/id_rsa
```

#### Добавление SSH-ключа на GitHub

Для добавления ключа надо его скопировать.

Например, таким образом можно отобразить ключ для копирования:

```shellsession
$ cat ~/.ssh/id_rsa.pub
```

После копирования надо перейти на GitHub. Находясь на любой странице, в правом верхнем углу нажмите на картинку Вашего профиля и в выпадающем списке выберите Settings. Затем в настройках на левой панели надо выбрать пункт "SSH and GPG keys". После надо нажать "New SSH key" и в поле "Title", написать название ключа (например "Home"), а в поле "Key" вставить содержимое, которое было скопировано из файла ~/.ssh/id_rsa.pub. Если GitHub запросит пароль, введите пароль своего аккаунта GitHub.

Чтобы проверить, всё ли прошло успешно, попробуйте выполнить команду локально:

```shellsession
$ ssh -T git@github.com
```

Вывод будет таким:

```
Hi username! You've successfully authenticated, but GitHub does not provide shell access.
```

Теперь Вы готовы работать с Git и GitHub.
